Overview

Observability is crucial for maintaining AI systems in production. It provides visibility into model performance, system health, and business impact, enabling proactive issue detection and resolution.

Three Pillars of Observability

Metrics (what), Logs (why), and Traces (where) provide comprehensive visibility into your AI system's behavior and performance.

Metrics

Quantitative measurements of system performance and behavior

Logs

Detailed records of events and system activities

Traces

Request flow through distributed systems

Metrics Collection

Key AI Model Metrics

Performance Metrics

  • • Prediction latency (P50, P95, P99)
  • • Throughput (requests/sec)
  • • Model accuracy/f1-score
  • • Resource utilization (CPU, GPU, Memory)

Business Metrics

  • • Prediction confidence scores
  • • Model drift indicators
  • • Feature distribution changes
  • • User engagement metrics

Prometheus Integration

from prometheus_client import Counter, Histogram, Gauge, start_http_server
import time
import random

# Define metrics
prediction_counter = Counter('model_predictions_total', 'Total number of predictions')
prediction_latency = Histogram('model_prediction_duration_seconds', 'Prediction latency')
model_accuracy = Gauge('model_accuracy_score', 'Current model accuracy')
active_requests = Gauge('model_active_requests', 'Number of active requests')

class AIModel:
    def __init__(self):
        self.accuracy = 0.95
        
    @prediction_latency.time()
    def predict(self, data):
        active_requests.inc()
        try:
            # Simulate prediction
            time.sleep(random.uniform(0.1, 0.5))
            prediction_counter.inc()
            
            # Update accuracy occasionally
            if random.random() < 0.1:
                self.accuracy = max(0.8, self.accuracy + random.uniform(-0.05, 0.05))
                model_accuracy.set(self.accuracy)
                
            return {"prediction": random.choice([0, 1]), "confidence": random.random()}
        finally:
            active_requests.dec()

# Start Prometheus metrics server
start_http_server(8001)

# Create model instance
model = AIModel()

# Simulate predictions
while True:
    result = model.predict([1, 2, 3])
    time.sleep(0.5)

Metric Design Principles

Focus on actionable metrics that indicate system health and business impact. Use percentiles (P50, P95, P99) rather than averages for latency measurements.

Structured Logging

AI-Specific Log Events

import logging
import json
from datetime import datetime
from pythonjsonlogger import jsonlogger

# Configure structured logging
logHandler = logging.StreamHandler()
formatter = jsonlogger.JsonFormatter()
logHandler.setFormatter(formatter)
logger = logging.getLogger()
logger.addHandler(logHandler)
logger.setLevel(logging.INFO)

class AIModelLogger:
    def __init__(self, model_name, version):
        self.model_name = model_name
        self.version = version
        self.logger = logger
    
    def log_prediction(self, input_data, prediction, confidence, latency):
        self.logger.info("prediction_made", extra={
            "model_name": self.model_name,
            "model_version": self.version,
            "prediction": prediction,
            "confidence": confidence,
            "latency_ms": latency * 1000,
            "input_hash": hash(str(input_data)),
            "timestamp": datetime.utcnow().isoformat()
        })
    
    def log_model_drift(self, drift_score, threshold):
        self.logger.warning("model_drift_detected", extra={
            "model_name": self.model_name,
            "drift_score": drift_score,
            "threshold": threshold,
            "action_required": drift_score > threshold,
            "timestamp": datetime.utcnow().isoformat()
        })
    
    def log_error(self, error, context):
        self.logger.error("model_error", extra={
            "model_name": self.model_name,
            "error_type": type(error).__name__,
            "error_message": str(error),
            "context": context,
            "timestamp": datetime.utcnow().isoformat()
        })

# Usage example
model_logger = AIModelLogger("sentiment_classifier", "v1.2.0")

# Log a prediction
model_logger.log_prediction(
    input_data="Great product!",
    prediction="positive",
    confidence=0.95,
    latency=0.150
)

Logging Best Practices

Avoid logging sensitive data (PII, credentials). Use correlation IDs to trace requests across distributed systems. Implement log retention policies.

Distributed Tracing

OpenTelemetry Integration

Track requests through your AI pipeline with distributed tracing for performance optimization. 

from opentelemetry import trace
from opentelemetry.exporter.otlp.proto.grpc.trace_exporter import OTLPSpanExporter
from opentelemetry.sdk.trace import TracerProvider
from opentelemetry.sdk.trace.export import BatchSpanProcessor
from opentelemetry.instrumentation.requests import RequestsInstrumentor
import time

# Configure OpenTelemetry
trace.set_tracer_provider(TracerProvider())
tracer = trace.get_tracer(__name__)

# Configure exporter
otlp_exporter = OTLPSpanExporter(
    endpoint="otel-collector:4317",
    insecure=True
)

# Add span processor
span_processor = BatchSpanProcessor(otlp_exporter)
trace.get_tracer_provider().add_span_processor(span_processor)

# Instrument HTTP requests
RequestsInstrumentor().instrument()

class AIModelWithTracing:
    def __init__(self, model_name):
        self.model_name = model_name
    
    @tracer.start_as_current_span("preprocess_data")
    def preprocess(self, raw_data):
        # Simulate preprocessing
        time.sleep(0.1)
        return {"processed": raw_data}
    
    @tracer.start_as_current_span("model_inference")
    def predict(self, processed_data):
        # Add model attributes to span
        span = trace.get_current_span()
        span.set_attribute("model.name", self.model_name)
        span.set_attribute("model.version", "1.0.0")
        
        # Simulate model inference
        time.sleep(0.3)
        
        # Add prediction attributes
        prediction = {"class": "positive", "confidence": 0.95}
        span.set_attribute("prediction.confidence", prediction["confidence"])
        
        return prediction
    
    @tracer.start_as_current_span("postprocess_result")
    def postprocess(self, prediction):
        # Simulate postprocessing
        time.sleep(0.05)
        return {"final_result": prediction}

# Usage with tracing
model = AIModelWithTracing("sentiment_classifier")

with tracer.start_as_current_span("prediction_pipeline") as pipeline_span:
    # Add pipeline metadata
    pipeline_span.set_attribute("pipeline.id", "12345")
    pipeline_span.set_attribute("user.id", "user_789")
    
    # Execute pipeline
    raw_data = "This product is amazing!"
    processed = model.preprocess(raw_data)
    prediction = model.predict(processed)
    result = model.postprocess(prediction)
    
    print(f"Final result: {result}")

Intelligent Alerting

Prometheus Alert Rules

# alert_rules.yml
groups:
- name: ai_model_alerts
  rules:
  - alert: HighPredictionLatency
    expr: histogram_quantile(0.95, model_prediction_duration_seconds_bucket) > 1.0
    for: 5m
    labels:
      severity: warning
    annotations:
      summary: "High prediction latency detected"
      description: "95th percentile latency is {{ $value }}s for model {{ $labels.model_name }}"

  - alert: ModelAccuracyDrop
    expr: model_accuracy_score < 0.85
    for: 2m
    labels:
      severity: critical
    annotations:
      summary: "Model accuracy dropped below threshold"
      description: "Model {{ $labels.model_name }} accuracy is {{ $value }}"

  - alert: HighErrorRate
    expr: rate(model_predictions_total[5m]) - rate(model_predictions_total[5m]) > 0.1
    for: 3m
    labels:
      severity: warning
    annotations:
      summary: "High error rate detected"
      description: "Error rate is {{ $value }} for model {{ $labels.model_name }}"

  - alert: ModelDriftDetected
    expr: model_drift_score > 0.8
    for: 1m
    labels:
      severity: critical
    annotations:
      summary: "Model drift detected"
      description: "Drift score {{ $value }} exceeds threshold for {{ $labels.model_name }}"

Alertmanager Configuration

# alertmanager.yml
global:
  smtp_smarthost: 'localhost:587'
  smtp_from: 'alerts@company.com'

route:
  group_by: ['alertname']
  group_wait: 10s
  group_interval: 10s
  repeat_interval: 1h
  receiver: 'web.hook'
  routes:
  - match:
      severity: critical
    receiver: critical_alerts
  - match:
      severity: warning
    receiver: warning_alerts

receivers:
- name: 'critical_alerts'
  email_configs:
  - to: 'oncall@company.com'
    subject: 'CRITICAL: {{ range .Alerts }}{{ .Annotations.summary }}{{ end }}'
    body: |
      {{ range .Alerts }}
      Alert: {{ .Annotations.summary }}
      Description: {{ .Annotations.description }}
      {{ end }}
  slack_configs:
  - api_url: 'https://hooks.slack.com/services/YOUR/SLACK/WEBHOOK'
    channel: '#ai-alerts'
    title: 'Critical AI Model Alert'
    text: '{{ range .Alerts }}{{ .Annotations.summary }}{{ end }}'

- name: 'warning_alerts'
  email_configs:
  - to: 'ai-team@company.com'
    subject: 'AI Model Warning'
    body: '{{ range .Alerts }}{{ .Annotations.description }}{{ end }}'

Alerting Best Practices

Use meaningful thresholds, avoid alert fatigue with proper severity levels, and include actionable information in alert descriptions.
