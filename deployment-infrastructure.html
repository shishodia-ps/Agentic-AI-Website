Containerization

Docker for AI Models

Containerize your AI models with all dependencies for consistent deployment across environments. 

# Dockerfile for a PyTorch model
FROM python:3.9-slim

WORKDIR /app

# Install system dependencies
RUN apt-get update && apt-get install -y \
    gcc \
    g++ \
    && rm -rf /var/lib/apt/lists/*

# Copy requirements and install Python dependencies
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# Copy model and application code
COPY model/ ./model/
COPY app.py .

# Expose port
EXPOSE 8000

# Run the application
CMD ["python", "app.py"]
# Build and run the container
docker build -t ai-model:latest .
docker run -p 8000:8000 ai-model:latest

Multi-stage Builds

Optimize container size with multi-stage builds for production deployment. 

# Multi-stage Dockerfile
FROM python:3.9-slim as builder

WORKDIR /app
COPY requirements.txt .
RUN pip install --user -r requirements.txt

FROM python:3.9-slim
WORKDIR /app

# Copy installed packages from builder stage
COPY --from=builder /root/.local /root/.local
COPY model/ ./model/
COPY app.py .

# Make sure scripts in .local are usable
ENV PATH=/root/.local/bin:$PATH

EXPOSE 8000
CMD ["python", "app.py"]

Best Practice

Use specific base image versions and multi-stage builds to reduce container size and improve security by minimizing the attack surface.

Kubernetes Deployment

Deploy and manage containerized AI models at scale with Kubernetes. 

# deployment.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: ai-model-deployment
spec:
  replicas: 3
  selector:
    matchLabels:
      app: ai-model
  template:
    metadata:
      labels:
        app: ai-model
    spec:
      containers:
      - name: ai-model
        image: ai-model:latest
        ports:
        - containerPort: 8000
        resources:
          requests:
            memory: "2Gi"
            cpu: "1"
          limits:
            memory: "4Gi"
            cpu: "2"
        livenessProbe:
          httpGet:
            path: /health
            port: 8000
          initialDelaySeconds: 30
          periodSeconds: 10
# service.yaml
apiVersion: v1
kind: Service
metadata:
  name: ai-model-service
spec:
  selector:
    app: ai-model
  ports:
    - protocol: TCP
      port: 80
      targetPort: 8000
  type: LoadBalancer

Serverless Deployment

AWS Lambda Function

Deploy AI models as serverless functions for event-driven, auto-scaling applications. 

# lambda_function.py
import json
import boto3
import torch
from transformers import AutoModelForSequenceClassification, AutoTokenizer

# Initialize model (outside handler for cold start optimization)
model_name = "distilbert-base-uncased-finetuned-sst-2-english"
tokenizer = AutoTokenizer.from_pretrained(model_name)
model = AutoModelForSequenceClassification.from_pretrained(model_name)

def lambda_handler(event, context):
    try:
        # Parse input
        body = json.loads(event['body'])
        text = body['text']
        
        # Tokenize and predict
        inputs = tokenizer(text, return_tensors="pt", truncation=True, padding=True)
        with torch.no_grad():
            outputs = model(**inputs)
            predictions = torch.nn.functional.softmax(outputs.logits, dim=-1)
        
        # Return results
        return {
            'statusCode': 200,
            'headers': {
                'Content-Type': 'application/json',
                'Access-Control-Allow-Origin': '*'
            },
            'body': json.dumps({
                'sentiment': 'positive' if predictions[0][1] > predictions[0][0] else 'negative',
                'confidence': float(max(predictions[0]))
            })
        }
    except Exception as e:
        return {
            'statusCode': 500,
            'body': json.dumps({'error': str(e)})
        }
# serverless.yml
service: ai-model-lambda

provider:
  name: aws
  runtime: python3.9
  memorySize: 3008
  timeout: 30
  environment:
    TRANSFORMERS_CACHE: /tmp/.cache
  
functions:
  predict:
    handler: lambda_function.lambda_handler
    events:
      - http:
          path: predict
          method: post
          cors: true
    layers:
      - arn:aws:lambda:region:account:layer:pytorch-layer:1

Cold Start Optimization

  • Keep models in memory outside the handler function
  • Use provisioned concurrency for critical functions
  • Optimize container size and dependencies
  • Use Lambda layers for shared dependencies

Serverless Best Practices

Serverless is ideal for variable workloads and event-driven architectures, but consider container-based solutions for sustained high-throughput applications.

TensorFlow Lite Deployment

# Convert model to TensorFlow Lite
import tensorflow as tf

# Load the trained model
model = tf.keras.models.load_model('model.h5')

# Convert to TensorFlow Lite
converter = tf.lite.TFLiteConverter.from_keras_model(model)
converter.optimizations = [tf.lite.Optimize.DEFAULT]
tflite_model = converter.convert()

# Save the converted model
with open('model.tflite', 'wb') as f:
    f.write(tflite_model)
# Deploy on edge device (Raspberry Pi)
import tensorflow as tf
import numpy as np

# Load the TensorFlow Lite model
interpreter = tf.lite.Interpreter(model_path='model.tflite')
interpreter.allocate_tensors()

# Get input and output tensors
input_details = interpreter.get_input_details()
output_details = interpreter.get_output_details()

# Make prediction
def predict(input_data):
    interpreter.set_tensor(input_details[0]['index'], input_data)
    interpreter.invoke()
    return interpreter.get_tensor(output_details[0]['index'])

# Example usage
input_data = np.array([[1.0, 2.0, 3.0]], dtype=np.float32)
prediction = predict(input_data)
print(f"Prediction: {prediction}")

Deployment Best Practices

CI/CD Pipeline

# .github/workflows/deploy.yml
name: Deploy AI Model

on:
  push:
    branches: [ main ]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
    - uses: actions/checkout@v2
    - name: Set up Python
      uses: actions/setup-python@v2
      with:
        python-version: '3.9'
    - name: Install dependencies
      run: |
        pip install -r requirements.txt
        pip install pytest
    - name: Run tests
      run: pytest tests/

  build-and-deploy:
    needs: test
    runs-on: ubuntu-latest
    steps:
    - uses: actions/checkout@v2
    - name: Build Docker image
      run: docker build -t ai-model:${{ github.sha }} .
    - name: Deploy to Kubernetes
      run: |
        kubectl set image deployment/ai-model-deployment \
          ai-model=ai-model:${{ github.sha }} \
          --record

Health Checks & Monitoring

Health Check Endpoint

@app.route('/health')
def health_check():
    try:
        # Check model availability
        model.predict([[1, 2, 3]])
        return jsonify({'status': 'healthy'}), 200
    except Exception as e:
        return jsonify({'status': 'unhealthy', 'error': str(e)}), 500
